## Verification Checklist

Run these commands to verify sensitive files won't be committed:

```bash
# Check if .env is properly ignored
git check-ignore -v backend/.env

# List all ignored files
git status --ignored

# Check for accidentally staged sensitive files
git diff --cached --name-only

# Look for common secrets in staged files (pre-commit check)
git diff --cached | grep -i "password\|api_key\|secret" || echo "No patterns found"

# Verify no .env files will be committed
git ls-files | grep -i ".env" | grep -v ".env.example" || echo "No .env files staged"

# Check for untracked potentially sensitive files
git ls-files --others --exclude-standard | grep -E ".env|\.key|\.pem|secrets" || echo "None found"
```

## Best Practices for Team

### Before Committing
1. ✅ Never manually create `.env` files in the repository root
2. ✅ Only use `.env.example` as templates
3. ✅ Always use `cp .env.example .env` locally
4. ✅ Review `.gitignore` when adding new environments

### Local Development
```bash
# Setup local environment
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env

# Edit with your local values (not committed)
vi backend/.env
vi frontend/.env
```

### CI/CD Pipeline
Set environment variables in CI/CD system (GitHub Actions, GitLab CI, Jenkins):
```yaml
# GitHub Actions example
env:
  SECRET_KEY: ${{ secrets.SECRET_KEY }}
  DATABASE_URL: ${{ secrets.DATABASE_URL }}
  ADMIN_PASSWORD: ${{ secrets.ADMIN_PASSWORD }}
```

## Secrets Scanning

### Automated Checks (Recommended)
Install pre-commit hooks to prevent secret leaks:

```bash
# Install pre-commit
pip install pre-commit

# Create .pre-commit-config.yaml
cat > .pre-commit-config.yaml << 'EOF'
repos:
  - repo: https://github.com/gitleaks/gitleaks
    rev: v8.18.0
    hooks:
      - id: gitleaks
EOF

# Install hooks
pre-commit install

# Test it
pre-commit run --all-files
```

Or use `truffleHog`:
```bash
pip install truffle-hog
trufflehog filesystem . --json
```
